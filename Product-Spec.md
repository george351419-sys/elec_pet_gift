## 1. 产品概述

### 1.1 产品定位

一款帮助父母为孩子定制专属 AI 陪伴宠物的网页应用。父母通过自然语音访谈向 AI 传递对孩子的了解，AI 将这些记忆赋能给一只会说话、有情绪的 3D 电子宠物；孩子则通过实时语音与宠物对话，宠物能记住每次聊天内容，像真正陪伴他成长的朋友。

### 1.2 目标用户

- **设置者（家长）**：有 4–8 岁孩子的父母，希望给孩子一个安全、懂孩子的数字陪伴，在自己不在身边时仍能安抚和陪伴孩子。
- **使用者（孩子）**：4–8 岁儿童，不需要识字，直接说话就能与宠物互动。

**典型使用场景：**
1. 睡前仪式：孩子对着宠物讲今天的事情，宠物帮助入睡
2. 父母不在家：孩子独处时有陪伴，宠物能回应情绪
3. 情绪安抚：孩子哭闹或害怕时，宠物用家长预设的方式安慰
4. 分享小秘密：孩子跟宠物说只属于它们的故事
5. 好奇问答：孩子随意提问，宠物按孩子能理解的方式回答

### 1.3 核心价值

- 对家长：不需要填表格，聊天就能完成孩子档案；宠物真正"懂"这个孩子，而不是通用的 AI 助手
- 对孩子：宠物有名字、有记忆、认识自己，不会遗忘上次聊的内容

## 2. 功能需求

### 2.1 Tab 结构

应用分两个 Tab，均在同一页面切换：

| Tab | 标签名 | 主要用户 | 核心功能 |
|-----|--------|----------|----------|
| Tab 1 | 家长设置 | 家长 | 语音访谈 + 档案管理 + 宠物选择 |
| Tab 2 | 宠物陪伴 | 孩子 | 与宠物实时全双工语音对话 |

---

### 2.2 Tab 1：家长设置

#### 功能 A：实时语音访谈

- **描述**：复用已有 `full-duplex-voice` 模块，`mode=parent_onboarding`。AI 扮演访谈引导师"小颖"，通过自然对话收集孩子信息，不要求填表。
- **输入**：家长语音
- **输出**：实时字幕 + 对话记录（访谈结束后可提取结构化档案）
- **触发**：点击"开始语音访谈"按钮
- **结束后**：将完整对话记录 + AI 自动提取的结构化字段一起保存
- **再次访谈**：可随时点击"继续补充"重新进入访谈，追加记忆

#### 功能 B：孩子档案管理

访谈结束后，AI 从对话中自动提取并填入以下结构化字段，家长可手动修改：

| 字段 | 类型 | 说明 |
|------|------|------|
| 孩子昵称 | 文本 | 宠物叫孩子的名字 |
| 年龄 | 数字 | 影响宠物语言复杂度 |
| 性格特点 | 标签（多选） | 活泼/安静/好奇/敏感/爱笑 等 |
| 喜欢的事 | 文本 | 宠物主动聊的话题 |
| 害怕的事 | 文本 | 宠物会主动回避或安抚 |
| 安抚方式 | 文本 | 孩子哭闹时宠物如何应对 |
| 亲子回忆 | 文本（长） | 1–3 段特别的家庭故事 |
| 鼓励方式 | 文本 | 孩子受挫时宠物如何鼓励 |
| 宠物昵称 | 文本 | 孩子给宠物起的名字（可选）|

- 字段填完后显示"保存档案"按钮
- 保存目标：`localStorage['magic_pet_profile']`

#### 功能 C：宠物形象选择

- **描述**：提供 5 种预设 3D 风格宠物，家长为孩子选一只
- **宠物列表**：英国短毛猫、美国短毛猫、泰迪（贵宾犬）、柴犬、金毛猎犬
- **展示形式**：横向卡片列表，每张卡片含 3D 宠物预览图 + 宠物名 + 简短性格描述
- **选中状态**：卡片高亮边框 + 底部"已选择"标记
- **保存目标**：`localStorage['magic_pet_selected']`

---

### 2.3 Tab 2：宠物陪伴

#### 功能 D：宠物主界面

- **未配置状态**：检测到 localStorage 无档案，显示提示"请让家长先完成设置"，禁用对话按钮
- **已配置状态**：显示所选宠物的 3D 形象居中，宠物昵称显示在图像上方

#### 功能 E：全双工实时语音对话

- **描述**：复用 `full-duplex-voice` 模块，`mode=child_pet`，将孩子档案 + 历史记忆摘要作为 `context` 注入
- **输入**：孩子语音
- **输出**：宠物合成语音（实时流式播放）+ 双方字幕
- **控件**：开始对话 / 静音 / 打断 / 结束
- **全双工**：孩子说话时宠物不会被打断（除非孩子点"打断"按钮）

#### 功能 F：宠物说话动画

- **触发条件**：远端音量（`remoteLevel`）> 0.05
- **动画效果**：宠物嘴部开合动画（说话态），静默时回到 idle 态
- **实现**：根据 `remoteLevel` 数值驱动 CSS animation 或帧序列切换，平滑过渡

#### 功能 G：字幕显示

- 宠物说话字幕：显示在宠物形象下方，字体偏大，儿童友好
- 孩子说话字幕：显示在底部输入区域，灰色小字
- 最近 4 轮字幕可见，旧字幕上移淡出

---

### 2.4 记忆系统

#### 记忆分层架构

| 层级 | 内容 | 存储方式 | 检索方式 |
|------|------|----------|----------|
| L1：结构化档案 | 家长访谈提取的固定字段 | localStorage JSON | 直接读取，每次注入 system prompt |
| L2：重要事实 | 孩子聊天中说出的重要信息（喜欢/害怕/新愿望等） | localStorage JSON 数组，含关键词 | 关键词匹配优先，全量注入（demo 阶段） |
| L3：对话摘要 | 历史对话的压缩摘要 | localStorage JSON 数组 | 最近 3 条摘要注入 system prompt |

#### 记忆更新规则

- **每次对话结束时**，调用后端接口（或前端直接调用大模型 API），对本次对话做两件事：
  1. **事实提取**：提取孩子说出的重要信息，写入 L2，已有相同 key 则更新（ADD/UPDATE）
  2. **对话压缩**：本次对话 > 5 轮时，生成一段摘要（100 字以内）写入 L3；≤5 轮则保留原始对话

- **每次对话开始时**，构建注入 context 的 system prompt：
  ```
  [孩子档案] L1 结构化字段全量
  [重要记忆] L2 事实列表（最多 20 条）
  [近期对话摘要] L3 最近 3 条摘要
  ```

- **demo 阶段**：记忆提取用同一个 Doubao 模型 API 调用完成，无需额外向量库

## 3. 用户流程

### 3.1 家长设置流程（首次）

```
1. 打开网页 → 默认进入「家长设置」Tab
2. 点击「开始语音访谈」→ 检查麦克风权限 → 加入 RTC 房间
3. AI 小颖通过语音与家长聊天（约 5–10 分钟）
4. 家长点击「结束访谈」→ 页面显示 AI 整理的结构化档案预览
5. 家长逐字段确认 / 修改 → 点击「保存档案」
6. 页面下方显示宠物选择区 → 家长选择一只宠物 → 点击「确认选择」
7. 提示「宠物已准备好，可以给孩子了」→ 家长可以切换到「宠物陪伴」Tab 给孩子
```

### 3.2 孩子对话流程

```
1. 进入「宠物陪伴」Tab
2. 看到选好的 3D 宠物，显示宠物昵称
3. 点击「开始对话」→ 宠物播放欢迎语音 + 嘴部动画
4. 孩子随时说话 → 字幕实时更新
5. 宠物回应 → 嘴部动画 + 字幕
6. 孩子可随时点「打断」让宠物停下
7. 点「结束」→ 宠物说再见 → 后台触发记忆更新
```

### 3.3 家长补充记忆流程

```
1. 进入「家长设置」Tab → 点击「继续补充」
2. 重新进入语音访谈 → 追加新内容（上次档案在 context 里）
3. 访谈结束 → 增量更新档案字段
```

### 3.4 异常流程

| 异常 | 触发条件 | 处理 |
|------|----------|------|
| 麦克风权限拒绝 | 用户拒绝授权 | 显示引导截图，提示如何在浏览器设置中开启 |
| RTC 连接失败 | 网络或配置问题 | 显示错误提示 + 重试按钮 |
| 档案未配置 | Tab 2 无 localStorage 数据 | 显示引导提示，禁用对话按钮 |
| 宠物无响应 | 远端音频超时未到达 | 显示"宠物在思考中…"状态，15s 后提示重试 |

## 5. AI 能力配置

| AI 能力 | 用途 | 使用时机 |
|---------|------|----------|
| 豆包 S2S 实时语音 | 全双工语音理解 + 语音合成 | 家长访谈 + 孩子对话 |

| DeepSeek Chat API（sk-e694f0…） | 访谈结束后提取结构化字段 + 对话压缩摘要 + 事实提取 | 每次会话结束时 |

## 6. 技术需求

### 6.1 技术栈

| 层 | 技术 |
|----|------|
| 前端框架 | React 18 + TypeScript + Vite |
| 样式 | CSS Modules / Tailwind（选一） |
| 语音模块 | 复用 `@magic-pet/full-duplex-voice`（已实现） |
| 宠物动画 | PNG 分帧 + CSS animation（idle / talking 两态） |
| 后端 | Express + TypeScript（复用现有 server） |
| 数据存储 | localStorage（demo 阶段） |
| 部署 | 本地 dev server（demo 验证阶段） |

### 6.2 数据结构

```typescript
// localStorage['magic_pet_profile']
type PetProfile = {
  nickname: string        // 孩子昵称
  age: number
  personality: string[]   // 性格标签
  likes: string
  fears: string
  comfort: string
  memories: string
  encouragement: string
  petName: string         // 宠物昵称
  petType: 'british_shorthair' | 'american_shorthair' | 'teddy' | 'shiba' | 'golden'
  createdAt: number
  updatedAt: number
}

// localStorage['magic_pet_facts']
type FactEntry = {
  key: string             // 事实关键词
  value: string           // 具体内容
  updatedAt: number
}

// localStorage['magic_pet_summaries']
type ConversationSummary = {
  summary: string         // 压缩后的对话摘要（≤100字）
  sessionDate: string
  turnCount: number
}
```

### 6.3 性能要求

- 语音延迟：端到端 < 1s（依赖豆包 S2S，非本地可控）
- 首屏加载：< 3s（3D 资源异步加载，不阻塞主界面）
- localStorage 记忆总量：< 500KB（对话摘要定期压缩，最多保留 20 条摘要）
